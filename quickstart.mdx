# Quickstart

Get PinchTab running and make your first API call to read a webpage in minutes.

## Prerequisites

Before you begin, make sure you have:

- PinchTab installed ([Installation guide](/installation))
- Google Chrome and Xvfb installed
- Chrome wrapper script created

## Start PinchTab

<Steps>
  <Step title="Start Xvfb display server">
    Ensure Xvfb is running to provide a virtual display:

    ```bash
    pgrep Xvfb || (Xvfb :99 -screen 0 1920x1080x24 &>/dev/null &)
    ```
  </Step>

  <Step title="Launch PinchTab dashboard">
    Start PinchTab in dashboard mode to manage multiple profiles:

    ```bash
    DISPLAY=:99 \
    CHROME_BINARY=~/.pinchtab/chrome-wrapper.sh \
    CHROME_FLAGS="--no-sandbox --disable-gpu" \
    nohup pinchtab dashboard > /tmp/pinchtab-dash.log 2>&1 &
    ```

    The dashboard will be available at `http://localhost:9867`.
  </Step>

  <Step title="Launch a browser profile">
    Create and launch your first browser profile:

    ```bash
    curl -s -X POST http://localhost:9867/instances/launch \
      -H "Content-Type: application/json" \
      -d '{"name":"default","port":"9868","headless":false}'
    ```

    This creates a profile named "default" running on port 9868.
  </Step>
</Steps>

## Make your first API call

Now that PinchTab is running, let's read a webpage:

<Steps>
  <Step title="Navigate to a webpage">
    Tell the browser to navigate to a URL:

    ```bash
    curl -s -X POST http://localhost:9868/navigate \
      -H "Content-Type: application/json" \
      -d '{"url":"https://example.com"}'
    ```
  </Step>

  <Step title="Extract readable text">
    Get the page's readable text content (most token-efficient):

    ```bash
    curl -s http://localhost:9868/text
    ```

    You'll get a JSON response like:

    ```json
    {
      "text": "Example Domain\nThis domain is for use in illustrative examples...",
      "title": "Example Domain",
      "url": "https://example.com/"
    }
    ```
  </Step>

  <Step title="Take a screenshot">
    Capture a screenshot of the page:

    ```bash
    curl -s http://localhost:9868/screenshot -o page.png
    ```

    This saves a PNG screenshot to `page.png`.
  </Step>
</Steps>

<Note>
The API for each profile runs on the port you specified when launching it. In this example, we used port 9868.
</Note>

## Quick reference

### Dashboard API (port 9867)

```bash
# List all profiles
curl -s http://localhost:9867/profiles

# List running instances
curl -s http://localhost:9867/instances

# Stop an instance
curl -s -X POST http://localhost:9867/instances/<id>/stop
```

### Instance API (port 9868)

```bash
# Navigate
curl -s -X POST http://localhost:9868/navigate \
  -H "Content-Type: application/json" \
  -d '{"url":"https://example.com"}'

# Get text
curl -s http://localhost:9868/text

# Get snapshot
curl -s http://localhost:9868/snapshot

# Screenshot
curl -s http://localhost:9868/screenshot -o screenshot.png

# Check health
curl -s http://localhost:9868/health
```

<Tip>
Use the `/text` endpoint by default — it's the most token-efficient way to read pages. Only use `/snapshot` when you need interactive elements, and `/screenshot` when you need visual data.
</Tip>
